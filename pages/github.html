Qu'est ce que Github ?


GitHub , exploité sous le nom de GitHub, Inc.) est un service web d'hébergement et de gestion de développement de logiciels, utilisant le logiciel de version Git. Ce site est développé en Ruby on rails et Erlang par Chris Wanstrath, PJ Hyett et Tom Preston-Werner. GitHub propose des comptes professionnels payants, ainsi que des comptes gratuits pour les projets de logiciels libres. Le site assure également un contrôle d'accès et des fonctionnalités destinées à la collaboration comme le suivi des bugs, les demandes de fonctionnalités, la gestion de tâches et un wiki pour chaque projet.

En avril 2016, GitHub a annoncé avoir dépassé les 14 millions d'utilisateurs et plus de 35 millions de dépôts de projets le plaçant comme le plus grand hébergeur de code source au monde.

Le nom GitHub est composé du mot « git » faisant référence à un système de contrôle de version open-source et le mot « hub » faisant référence au réseau social bâti autour du système Git, mais aussi à une plate-forme de correspondance qui est appelée en anglais un « hub ».

Son accessibilité et le partage avec d’autres utilisateurs font de cette plateforme l’outil indispensable dont il faudra vous équipez si vous souhaitez organiser le travail de groupe sur un projet.

Pour celles et ceux qui ne sont pas initiés a certaines pratiques ou vocabulaire informatique, voici une représentation visuel de l’utilisation classique de Github et simplifier à son maximum, afin d'en facilité sa compréhension.



Comme vous pourrez le constatez ci dessus, cette représentation visuel nous permet d’observer plusieurs choses. En effet votre projet est initialement crée par un ou plusieurs ordinateurs qui héberge eux meme les données du projet.

Prenons un exemple classique d’utilisation :
« 1 » , « 2 » et « 3 » souhaitent réaliser un seul et unique projet.
« 1 » , « 2 » et « 3 » vont chacun se connecter à la plateforme d’hébergement via leurs identifiants, nommé leurs projets, et ainsi le préparé a recevoir des données.
Ceci étant fait, « 1 » , « 2 » et « 3 » sont tous chacun prêt à exporter un segment de leurs travaux dans un seul et unique projet.
Pour cela ils vont chacun ordonner à leurs machine, Github, pousse moi les données contenue dans ma machine , vers mon projet GitHub que tu héberge.
Ainsi fait, les utilisateurs pourront à l’aide de commandes spécial, mettre à jour individuellement le projet tout en gardant la trace via un historique, et ainsi la possibilité de revenir sur certaines décisions.

Les commandes de Bases Git

Git config :

La commande suivante peut être utilisée pour définir le mail d’un utilisateur :


                                git config --global user.email jonathan@google.com

Git init :

Cette commande est utilisée pour créer un nouveau dépôt GIT : ( "dépôt" ou "repository" représentez vous un "dossier" / "folder"


                                git init

Git add :

La commande git add peut être utilisée pour ajouter un fichier récement modifier. La commande suivante ajoutera un fichier nommé temp.txt dans le dossier local situer dans la mémoire de votre ordinateur :


                                git add temp.txt

Git add . :

La commande git add . , peut être utilisée pour ajouter la totalité des fichiers récement modifier. La commande suivante ajoutera la totalité des fichier récement utiliser dans le dossier local situer dans la mémoire de votre ordinateur :


                                git add .
